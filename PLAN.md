## 🔒 ეტაპი 2: ავტორიზაცია, სესიები და Auth Guard (სამუშაო დრო: 1 დღე)
* [ ] **2.1. LocalStorage-ის ბაზისური ფუნქციები (`js/storage.js`):**
  * დაწერე დამხმარე ფუნქციები მონაცემების წასაკითხად და ჩასაწერად (`localStorage.getItem`, `JSON.parse`, `JSON.stringify`).
* [ ] **2.2. Sign Up გვერდის აწყობა და ვალიდაცია (`signup.html`, `js/auth.js`):**
  * ააწყვე ფორმა 5 ველით (Full Name, Email, Company, Password, Confirm Password).
  * `submit` ივენთზე გააკეთე `preventDefault()` და 6-ვე ვალიდაციის წესის ერთდროული შემოწმება.
  * დაამატე შეცდომების ვიზუალიზაცია წითელი ტექსტით ველის ქვეშ და `.input-error` კლასით.
  * წარმატებისას შექმენი მომხმარებლის ობიექტი, ჩაწერე `crm_users`-ში, გამოაჩინე მწვანე Toast და 1.5 წამში გადაამისამართე `index.html`-ზე.
* [ ] **2.3. Login გვერდის აწყობა და ლოგიკა (`index.html`, `js/auth.js`):**
  * ააწყვე ფორმა (Email, Password) და „Log In“ ღილაკი.
  * `submit`-ზე შეამოწმე ველების შევსება, შემდეგ კი `crm_users`-ში `find()` მეთოდით მოძებნე მომხმარებელი.
  * არასწორ მონაცემებზე გამოაჩინე ზოგადი შეცდომა: `Invalid email or password`.
  * წარმატებისას შექმენი `crm_session` ობიექტი და გადაიყვანე დეშბორდზე.
* [ ] **2.4. Auth Guard & Logout (`js/guard.js`):**
  * დაწერე გლობალური ფუნქცია, რომელიც ამოწმებს `crm_session`-ს.
  * თუ დაცულ გვერდზე სესია არაა -> გადაამისამართე `index.html`-ზე.
  * თუ საჯარო გვერდზე სესია არის -> გადაამისამართე `dashboard.html`-ზე.
  * ააწყვე ნავიგაციის ზოლი და `Logout` ღილაკი, რომელიც შლის სესიას და აბრუნებს ლოგინზე.

## 👥 ეტაპი 3: Clients გვერდის ბირთვი (CORE) (სამუშაო დრო: 1 დღე)
* [ ] **3.1. API-დან მონაცემების წამოღება (`js/clients.js`):**
  * გვერდის გახსნისას შეამოწმე `crm_clients` ბრაუზერში. 
  * თუ ცარიელია, გამოიძახე `GET https://dummyjson.com` (`async/await` და `try/catch` გამოყენებით).
  * წამოღებული მონაცემები `map()` მეთოდით გარდაქმენი `Client` ობიექტის სტრუქტურად და შეინახე `localStorage`-ში.
  * მონაცემების მოლოდინში გამოაჩინე „Loading clients...“.
* [ ] **3.2. სიის ვიზუალური რენდერი (DOM მანიპულაცია):**
  * შექმენი ფუნქცია `renderClients(list)`.
  * თითოეული კლიენტისთვის შექმენი ბარათი (ავატარი, სახელი, კომპანია, სტატუსის ბეჯი, deal value, Delete ღილაკი `data-id`-ით).
  * თუ სია ცარიელია, გამოაჩინე „No clients found.“
* [ ] **3.3. კლიენტის დამატება (Add Client Modal):**
  * შექმენი მოდალური ფანჯარა საჭირო ფორმის ველებით.
  * `submit`-ზე გაუკეთე ვალიდაცია (წესები იხ. PRD P4.4) და გაგზავნე იმიტირებული `POST` რექვესთი `https://dummyjson.com`.
  * სერვერისგან პასუხის მიღების შემდეგ, ახალი კლიენტი ჩაამატე მასივის დასაწყისში (`unshift`), შეინახე `localStorage`-ში, განაახლე ეკრანი და დახურე მოდალი.
* [ ] **3.4. კლიენტის წაშლა (Delete):**
  * Delete ღილაკზე დაჭერისას გამოიძახე ბრაუზერის `confirm()`.
  * თანხმობისას გაგზავნე `DELETE` რექვესთი.
  * მიუხედავად API პასუხისა (200 თუ 404), წაშალე ელემენტი `crm_clients` მასივიდან `filter()`-ით, შეინახე ბაზაში და დაარენდერე სია თავიდან.

## 📊 ეტაპი 4: Dashboard და FULL ფიჩერები (სამუშაო დრო: 1 დღე)
* [ ] **4.1. მისალმების ზოლი და ცოცხალი საათი (`js/dashboard.js`):**
  * სესიიდან ამოიღე მომხმარებლის სახელი, დაყავი `split(' ')[0]`-ით და გამოაჩინე.
  * `setInterval`-ით აამუშავე საათი (`toLocaleTimeString()` და `toLocaleDateString()`).
* [ ] **4.2. სტატისტიკის დათვლა და Pipeline Overview:**
  * `length`, `filter` და `reduce` მეთოდებით გამოთვალე 4 სტატისტიკური ბარათის მნიშვნელობა (Total Clients, Active Deals, Won Revenue ფორმატით, New This Week).
  * შექმენი Pipeline ბლოკები თითოეული სტატუსის რაოდენობით.
* [ ] **4.3. Recent Clients სია:**
  * `crm_clients` მასივი დასორტე `createdAt`-ით კლებადობით, აიღე პირველი 5 ელემენტი (`slice(0, 5)`) და დაარენდერე.
* [ ] **4.4. ფილტრაცია, ძებნა და სორტირება (Clients გვერდი - FULL):**
  * შექმენი ერთი ცენტრალური ფუნქცია `getVisibleClients()`, რომელიც საწყისი მასივის ასლზე თანმიმდევრობით გამოიყენებს: სტატუსის ჩიპის ფილტრს -> საძიებო ველის ტექსტს (`includes`) -> სორტირების წესს.
  * ნებისმიერ Toolbar ივენთზე (`input`, `click`, `change`) გამოიძახე ეს ფუნქცია და შედეგი გადაეცი `renderClients()`-ს.
* [ ] **4.5. სტატუსის ცვლილება უშუალოდ ბარათიდან:**
  * ბარათზე არსებული სტატუსის `select`-ით შეცვლისას (`change` ივენთი) განაახლე მონაცემი `localStorage`-ში.

## 📑 ეტაპი 5: კლიენტის დეტალები, პროფილი და თემები (სამუშაო დრო: 1 დღე)
* [ ] **5.1. დეტალების მოდალი და შენიშვნები (Notes):**
  * კლიენტის ბარათზე დაჭერისას (გარდა ღილაკებისა) გახსენი დეტალების მოდალი.
  * გამოაჩინე არსებული შენიშვნების სია (Notes).
  * დაამატე ახალი შენიშვნის ჩაწერის ფუნქციონალი (ვალიდაციით), რომელიც შეინახება კლიენტის `notes` მასივში.
* [ ] **5.2. Follow-up შეხსენება (Reminder):**
  * ღილაკზე „Remind me in 1 min“ დაჭერისას გაუშვი `setTimeout` 60000 მილიწამზე.
  * დროის გასვლის შემდეგ ამოაგდე გლობალური Toast შეტყობინება, იმის მიუხედავად, მოდალი ღიაა თუ დახურული.
* [ ] **5.3. Profile გვერდი (`profile.html`, `js/profile.js`):**
  * გამოაჩინე მიმდინარე მომხმარებლის ინფო (ინიციალების ავატარი, სახელი, იმეილი, კომპანია).
  * ფორმა ა): პროფილის რედაქტირება (Full Name, Company) + ვალიდაცია + მონაცემების განახლება `crm_users`-ში.
  * ფორმა ბ): პაროლის შეცვლა (Current, New, Confirm New) + ვალიდაციები და დაჰინტებული შეცდომის ტექსტები.
  * ღილაკი გ): Reset CRM Data — კლიენტების მასივის წაშლა `localStorage`-დან და თავიდან წამოღება API-დან.
* [ ] **5.4. Dark / Light თემის გლობალური მართვა:**
  * `js/storage.js`-ში დაწერე თემის შეცვლის ლოგიკა, რომელიც `body`-ზე ამატებს/აშორებს კლასს და ინახავს მნიშვნელობას `crm_theme`-ში.
  * ჩართე ეს ლოგიკა ყოველი გვერდის ჩატვირთვის საწყის ეტაპზე.
